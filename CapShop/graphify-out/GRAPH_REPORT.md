## 1. God Nodes (Highest Connectivity / Cross-Cutting)

These nodes are referenced by the most other nodes in the graph. Changing them has the widest blast radius.

| Rank | Node | Type | Connections | Why It's a God Node |
|------|------|------|-------------|---------------------|
| 1 | `CapShop.Shared` | Library | ALL 5 services + Gateway | Every service depends on it for ApiResponse, exceptions, middleware, messaging contracts |
| 2 | `ApiResponse<T>` | Class | ~30 controllers/handlers | Universal response wrapper used in every HTTP response across all services |
| 3 | `RabbitMqMessagePublisher` | Class | 4 services (Order, Admin, Catalog, Notification) | The single async messaging backbone; all saga events flow through it |
| 4 | `GlobalExceptionMiddleware` | Class | All 5 services + Gateway | Centralized error handling registered in every service pipeline |
| 5 | `axiosClient.ts` | Module | All 5 frontend API modules | Single Axios instance with JWT interceptor; all frontend HTTP calls go through it |
| 6 | `useAuthStore` | Zustand Store | 10+ frontend components/hooks | Auth state (token, user, role) consumed everywhere; also used by axiosClient |
| 7 | `JwtSettings` (config) | Config Section | Gateway + all 4 protected services | JWT secret/issuer/audience must be identical across Gateway, Order, Admin, Notification, Catalog |
| 8 | `SagaIntegrationEvents.cs` | Contracts File | Order + Admin + Catalog services | Defines all 6 saga event records that orchestrate the distributed checkout transaction |
| 9 | `AppRouter.tsx` | Component | All 20+ page components | Single routing tree; every page, guard, and layout is wired here |
| 10 | `docker-compose.yml` | Config | All 9 containers | Defines the entire runtime topology, env vars, health checks, and startup order |

## 2. Community Structure (Clusters)

The graph partitions into 7 natural communities based on import/dependency density.

### Community A — Auth Domain
**Centroid:** `AuthController` → `LoginCommandHandler` → `JwtTokenGenerator`
```
AuthController
  ├── RegisterUserCommandHandler  →  IUserRepository  →  UserRepository  →  AuthDbContext
  ├── LoginCommandHandler         →  IPasswordHasher   →  BcryptPasswordHasher
  ├── GetUserQueryHandler         →  IUserRepository
  ├── VerifyTwoFactorCommandHandler → IEmailService   →  EmailService
  ├── ManageTwoFactorCommandHandler
  ├── ForgotPasswordCommandHandler
  └── ResetPasswordCommandHandler
Domain: User, Address, UserRoles
Validators: SignupRequestValidator (FluentValidation)
```
**Boundary:** Auth service is the ONLY issuer of JWT tokens. All other services only validate.

---

### Community B — Catalog Domain
**Centroid:** `ProductsController` → query handlers → `IProductRepository`
```
ProductsController
  ├── GetProductsQueryHandler       →  IProductRepository  →  ProductRepository  →  CatalogDbContext
  ├── GetProductByIdQueryHandler
  ├── GetFeaturedProductsQueryHandler
  └── GetCategoriesQueryHandler     →  ICategoryRepository →  CategoryRepository
Background Consumers (IHostedService):
  ├── OrderCancelledConsumer        ← listens: orders.cancelled
  └── InventoryReservationConsumer  ← listens: saga.payment.completed
Domain: Product, Category
```
**Boundary:** Catalog is READ-ONLY from HTTP. All writes come via RabbitMQ (inventory reservation, cancellation restores).

---

### Community C — Order + Cart Domain
**Centroid:** `PlaceOrderCommandHandler` (highest internal fan-out)
```
CartController
  ├── AddToCartCommandHandler      →  ICartRepository
  ├── UpdateCartItemCommandHandler
  └── RemoveCartItemCommandHandler
OrdersController
  ├── SimulatePaymentCommandHandler →  IPaymentSimulationRepository
  ├── PlaceOrderCommandHandler      →  IOrderRepository + IMessagePublisher (publishes OrderCheckoutInitiatedIntegrationEvent)
  └── CancelOrderCommandHandler
  ├── GetCartQueryHandler
  ├── GetMyOrdersQueryHandler
  └── GetOrderByIdQueryHandler
Saga Consumers (IHostedService):
  ├── OrderSagaPaymentFailedConsumer    ← saga.payment.failed
  ├── OrderSagaInventoryReservedConsumer ← saga.inventory.reserved
  └── OrderSagaInventoryFailedConsumer  ← saga.inventory.failed
Domain: Cart, CartItem, Order, OrderItem, OrderStatusHistory, PaymentSimulation
```
**Boundary:** Order service is the saga ORCHESTRATOR — it initiates checkout and reacts to all saga outcomes.

---

### Community D — Admin Domain
**Centroid:** `AdminDbContext` (receives events from 3 different queues)
```
AdminProductsController  →  IAdminProductRepository  →  AdminProductRepository
AdminOrdersController    →  IAdminOrderRepository    →  AdminOrderRepository
DashboardController      →  IDashboardRepository     →  DashboardRepository
ReportsController        →  IReportsRepository       →  ReportsRepository + ReportExportService
AuditLog                 →  IAuditLogRepository      →  AuditLogRepository
Background Consumers:
  ├── OrderPlacedConsumer    ← orders.placed
  ├── PaymentSagaConsumer    ← saga.checkout.initiated
  └── RefundSagaConsumer     ← saga.refund.requested
Domain: AdminProduct, AdminOrder, DashboardMetrics, AuditLog
```
**Boundary:** Admin is purely a READ + AUDIT service. It never publishes events; it only consumes them.

---

### Community E — Notification Domain
**Centroid:** `OrderPlacedConsumer` → `EmailService`
```
NotificationsController  →  (read notifications)
Background Consumers:
  └── OrderPlacedConsumer  ← orders.placed  →  EmailService (SMTP)
Domain: Notification, EmailDeliveryStatus
```
**Boundary:** Notification is the most isolated service — no HTTP writes, no saga participation, email-only output.

---

### Community F — Frontend Feature Modules
**Centroid:** `axiosClient.ts` + `useAuthStore`
```
axiosClient.ts  ←  useAuthStore (token injection)
  ├── authApi.ts      →  /auth/*
  ├── catalogApi.ts   →  /catalog/*
  ├── orderApi.ts     →  /orders/*
  └── adminApi.ts     →  /admin/*

Feature modules (each owns: pages/ + hooks/ + components/):
  ├── auth/     →  authApi, useAuthStore, authStore
  ├── catalog/  →  catalogApi, product.types
  ├── cart/     →  orderApi, cartStore
  ├── checkout/ →  orderApi, cartStore, authStore
  ├── orders/   →  orderApi, order.types
  ├── admin/    →  adminApi, admin.types
  └── account/  →  authApi, authStore

Guards (route protection):
  ├── AdminGuard    →  useAuthStore (role === "Admin")
  ├── CustomerGuard →  useAuthStore (role === "Customer")
  └── GuestGuard    →  useAuthStore (!isAuthenticated)
```
