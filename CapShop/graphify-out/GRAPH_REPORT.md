## 1. God Nodes (Highest Connectivity / Cross-Cutting)

These nodes are referenced by the most other nodes. Changing them has the widest blast radius.

| Rank | Node | Type | Connections | Why It Is a God Node |
|------|------|------|-------------|----------------------|
| 1 | `CapShop.Shared` | Library | ALL 5 services + Gateway | Every service depends on it for ApiResponse, exceptions, middleware, messaging contracts |
| 2 | `ApiResponse<T>` | Class | ~35 controllers/handlers | Universal response wrapper used in every HTTP response across all services |
| 3 | `RabbitMqMessagePublisher` | Class | 4 services | The single async messaging backbone; all saga events flow through it |
| 4 | `GlobalExceptionMiddleware` | Class | All 5 services + Gateway | Centralized error handling registered in every service pipeline |
| 5 | `axiosClient.ts` | Module | All 5 frontend API modules | Single Axios instance with JWT interceptor; all gateway HTTP calls go through it |
| 6 | `useAuthStore` | Zustand Store | 12+ components/hooks | Auth state (token, user, role) consumed everywhere; also used by axiosClient |
| 7 | `useChatStore` | Zustand Store | 8 chat components + useChat hook | NEW: Central chat state machine, message history, context memory |
| 8 | `useChat` hook | React Hook | 13 chat components | NEW: Orchestrates all chat intents, API calls, state transitions |
| 9 | `intentClassifier.ts` | Module | useChat (sendMessage) | NEW: Every user message passes through this; determines which handler runs |
| 10 | `SagaIntegrationEvents.cs` | Contracts File | Order + Admin + Catalog | Defines all 6 saga event records for the distributed checkout transaction |
| 11 | `docker-compose.yml` | Config | All 10 containers | Defines the entire runtime topology including the new ai-service container |
| 12 | `dev-start.ps1` | Script | All 8 services + Ollama | NEW: Single entry point for local dev; starts everything in the correct order |

### 4c. Local Dev Startup Sequence

    dev-start.ps1
      Step 1: Check SQLEXPRESS, Redis :6379, RabbitMQ :5672
      Step 2: ollama serve + ollama pull llama3.2:1b
      Step 3: dotnet ef database update (x5 databases)
      Step 4: Start 5 .NET microservices (separate windows)
      Step 5: Start Gateway, wait for :5000
      Step 6: DataSeeder (if .capshop-seeded absent)
              Wait for /swagger/v1/swagger.json HTTP 200 (not just TCP)
              Write .capshop-seeded flag on success
      Step 7: uvicorn fastapi_app:app --port 8001, wait for :8001
      Step 8: npm run dev (Vite), wait for :5173
